---
title: "Recording API: Codegen Sessions and Code Parsing"
sidebarTitle: "Recording"
description: "Start and stop Playwright Codegen recording sessions to capture browser interactions, retrieve generated nodes, or parse existing Playwright code into flow nodes."
---

The Recording API wraps Playwright's built-in Codegen tool, letting you capture real browser interactions and instantly convert them into importable QA Flow nodes. You can start a recording session from any URL, stop it when done, and then either retrieve the raw TypeScript code or get the interactions as structured node definitions ready to drop into a project. Alternatively, if you already have Playwright TypeScript test code, the `/parse-code` endpoint converts it to nodes without a live browser session.

<Note>
  Recording sessions use an external browser process. If you are running QA Flow inside Docker, use the **interactive picker** endpoints (`/picker/interactive/*`) rather than `/record/start`, which requires a display server or GUI environment.
</Note>

---

## POST /api/record/start

Start a new Playwright Codegen recording session. Opens a browser at the specified URL and begins recording all user interactions. Returns a `sessionId` used to reference the session in subsequent calls.

**Authentication required:** Yes

### Request Body

<ParamField body="url" type="string" required>
  The starting URL the browser will navigate to when the recording session opens.
</ParamField>

### Response

<ResponseField name="sessionId" type="string">
  A unique identifier for the recording session. Pass this to all other `/record/*` endpoints.
</ResponseField>

<CodeGroup>
```bash cURL
curl -X POST http://localhost:3001/api/record/start \
  -H 'Authorization: Bearer <token>' \
  -H 'Content-Type: application/json' \
  -d '{"url": "https://app.example.com"}'
```

```json Response 200
{
  "sessionId": "sess-r1s2t3-..."
}
```
</CodeGroup>

## GET /api/record/status/:sessionId

Check the current status of an active recording session.

**Authentication required:** Yes

### Path Parameters

<ParamField path="sessionId" type="string" required>
  The session identifier returned by `POST /api/record/start`.
</ParamField>

### Response

<ResponseField name="sessionId" type="string">
  The session identifier.
</ResponseField>

<ResponseField name="status" type="string">
  Current state of the recording session. One of: `recording`, `stopped`.
</ResponseField>

<CodeGroup>
```bash cURL
curl http://localhost:3001/api/record/status/sess-r1s2t3 \
  -H 'Authorization: Bearer <token>'
```

```json Response 200
{
  "sessionId": "sess-r1s2t3",
  "status": "recording"
}
```
</CodeGroup>

## POST /api/record/stop/:sessionId

Stop an active recording session. The browser is closed and the recorded Playwright code is finalized and saved for retrieval.

**Authentication required:** Yes

### Path Parameters

<ParamField path="sessionId" type="string" required>
  The session identifier to stop.
</ParamField>

### Response

A success confirmation that the session has been stopped.

<CodeGroup>
```bash cURL
curl -X POST http://localhost:3001/api/record/stop/sess-r1s2t3 \
  -H 'Authorization: Bearer <token>'
```

```json Response 200
{
  "success": true,
  "sessionId": "sess-r1s2t3"
}
```
</CodeGroup>

## GET /api/record/code/:sessionId

Retrieve the raw Playwright TypeScript code generated by the recording session. Available after the session has been stopped.

**Authentication required:** Yes

### Path Parameters

<ParamField path="sessionId" type="string" required>
  The session identifier whose generated code to retrieve.
</ParamField>

### Response

<ResponseField name="code" type="string">
  The Playwright TypeScript test code captured during the recording session.
</ResponseField>

<CodeGroup>
```bash cURL
curl http://localhost:3001/api/record/code/sess-r1s2t3 \
  -H 'Authorization: Bearer <token>'
```

```json Response 200
{
  "code": "import { test, expect } from '@playwright/test';\n\ntest('recorded test', async ({ page }) => {\n  await page.goto('https://app.example.com');\n  await page.getByRole('button', { name: 'Sign In' }).click();\n  await page.getByLabel('Email').fill('user@example.com');\n  await page.getByLabel('Password').fill('password123');\n  await page.getByRole('button', { name: 'Submit' }).click();\n  await expect(page).toHaveURL('/dashboard');\n});\n"
}
```
</CodeGroup>

## GET /api/record/nodes/:sessionId

Convert the recorded session into QA Flow node definitions ready to import into a project. This is the primary endpoint for turning a recording into a reusable test flow without manually building the node graph.

**Authentication required:** Yes

### Path Parameters

<ParamField path="sessionId" type="string" required>
  The session identifier whose interactions to convert to nodes.
</ParamField>

### Response

<ResponseField name="nodes" type="NodeDefinition[]">
  Array of node definition objects ready for import into a QA Flow project.

  <Expandable title="NodeDefinition fields">
    <ResponseField name="id" type="string">Generated node ID.</ResponseField>
    <ResponseField name="type" type="string">Node type (e.g. `navigate`, `click`, `fill`, `assert`).</ResponseField>
    <ResponseField name="data" type="object">Node-specific configuration data (selector, URL, value, etc.).</ResponseField>
    <ResponseField name="position" type="object">Suggested `{ x, y }` position in the visual canvas.</ResponseField>
  </Expandable>
</ResponseField>

<CodeGroup>
```bash cURL
curl http://localhost:3001/api/record/nodes/sess-r1s2t3 \
  -H 'Authorization: Bearer <token>'
```

```json Response 200
{
  "nodes": [
    {
      "id": "node-gen-1",
      "type": "navigate",
      "data": { "url": "https://app.example.com" },
      "position": { "x": 100, "y": 100 }
    },
    {
      "id": "node-gen-2",
      "type": "click",
      "data": { "selector": "button[name='Sign In']" },
      "position": { "x": 100, "y": 200 }
    },
    {
      "id": "node-gen-3",
      "type": "fill",
      "data": { "selector": "label[for='Email']", "value": "user@example.com" },
      "position": { "x": 100, "y": 300 }
    }
  ]
}
```
</CodeGroup>

## POST /api/parse-code

Parse existing Playwright TypeScript test code into QA Flow node definitions without starting a browser session. Useful for importing existing Playwright tests you've written manually or copied from another source.

**Authentication required:** Yes

### Request Body

<ParamField body="code" type="string" required>
  Valid Playwright TypeScript test code to parse. Must follow Playwright's standard `test(...)` and `page.*()` API patterns.
</ParamField>

### Response

<ResponseField name="nodes" type="NodeDefinition[]">
  Array of node definition objects derived from the parsed Playwright code.
</ResponseField>

<CodeGroup>
```bash cURL
curl -X POST http://localhost:3001/api/parse-code \
  -H 'Authorization: Bearer <token>' \
  -H 'Content-Type: application/json' \
  -d '{
    "code": "import { test, expect } from '\''@playwright/test'\'';\n\ntest('\''login test'\'', async ({ page }) => {\n  await page.goto('\''https://app.example.com'\'');\n  await page.fill('\''#email'\'', '\''admin@qaflow.com'\'');\n  await page.fill('\''#password'\'', '\''admin123'\'');\n  await page.click('\''#login-btn'\'');\n});"
  }'
```

```json Response 200
{
  "nodes": [
    {
      "id": "node-p-1",
      "type": "navigate",
      "data": { "url": "https://app.example.com" },
      "position": { "x": 100, "y": 100 }
    },
    {
      "id": "node-p-2",
      "type": "fill",
      "data": { "selector": "#email", "value": "admin@qaflow.com" },
      "position": { "x": 100, "y": 200 }
    },
    {
      "id": "node-p-3",
      "type": "fill",
      "data": { "selector": "#password", "value": "admin123" },
      "position": { "x": 100, "y": 300 }
    },
    {
      "id": "node-p-4",
      "type": "click",
      "data": { "selector": "#login-btn" },
      "position": { "x": 100, "y": 400 }
    }
  ]
}
```
</CodeGroup>

## DELETE /api/recordings/:sessionId

Delete a specific recording session and its associated data from the server.

**Authentication required:** Yes

### Path Parameters

<ParamField path="sessionId" type="string" required>
  The session identifier to delete.
</ParamField>

### Response

Returns `204 No Content` on success.

```bash
curl -X DELETE http://localhost:3001/api/recordings/sess-r1s2t3 \
  -H 'Authorization: Bearer <token>'
```

---

## DELETE /api/recordings

Delete **all** recording sessions and their associated data from the server. Use with caution — this operation is irreversible.

**Authentication required:** Yes

### Response

Returns `204 No Content` on success.

```bash
curl -X DELETE http://localhost:3001/api/recordings \
  -H 'Authorization: Bearer <token>'
```

<Warning>
  This endpoint deletes every recording session on the server, not just your own. Ensure no other users have active or unsaved sessions before calling this endpoint.
</Warning>

---

## POST /api/recordings/cleanup

Remove stale or orphaned recording sessions — sessions where the browser process has exited or crashed but the session record was not properly cleaned up. Returns the number of sessions that were removed.

**Authentication required:** Yes

### Response

<ResponseField name="cleaned" type="number">
  The number of stale sessions that were removed.
</ResponseField>

<CodeGroup>
```bash cURL
curl -X POST http://localhost:3001/api/recordings/cleanup \
  -H 'Authorization: Bearer <token>'
```

```json Response 200
{
  "cleaned": 3
}
```
</CodeGroup>

<Tip>
  Schedule periodic calls to `/recordings/cleanup` in long-running deployments to prevent orphaned browser processes from consuming server memory.
</Tip>
